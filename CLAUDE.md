## Что это
Детское развивающее приложение **«Chisana kodomo»** (ちいさなこども — «маленькие
дети») для малышей **3–6 лет** (+ родитель). **Flutter + Flame**, только
**Android** (apk/aab), организация `com.auskraft`, applicationId
`com.auskraft.chisana_kodomo`.

**Бесплатно, без рекламы, без доната, офлайн** — это и правильная практика для
детских приложений. Бренд **тёплый и яркий** (не «ночная» тема Natsukashi Yoru):
кремовый фон, коралл/небо/солнце, маскот-зверушка (арт сгенерим позже).

UX **«без проигрышей»**: большие кнопки, голосовые подсказки (TTS), мгновенная
похвала + звёздочки/наклейки, ошибка → мягкое «попробуй ещё». **Никаких таймеров
и Game Over.** Приоритет — мягкий детский «сок»: тёплое поощрение, не
дофамин-аркада.

  (как в natsukashi_tanks/_yoru). Это **не машина пользователя** (он собирает в
  Android Studio). Агент верифицирует только через **`flutter analyze` +
  `flutter test`** (работают). `flutter create`, `pub get`, `dart run`,
  генераторы — работают.

  архитектуры core/ + 4 слоя) и `A:\StudioProjects\natsukashi_yoru` (14 мини-игр).
  Glob/Grep до соседних папок **не достают**, request_directory заблокирован →
  читать `Read` по абсолютному пути, искать `Bash cd A:\StudioProjects\... && grep`.
- **Коммитим по ходу** прямо в `main`, без напоминаний. В сообщениях коммитов —
  строка `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.
  Пуш — по просьбе владельца.
- **Пуш больших ассетов из агента НЕ проходит** (фоны ~16 МБ → канал рвётся,
  `Recv failure`/таймаут). **origin может отставать от local** — проверь
  `git log --oneline origin/main..main`. Синхронизирует GitHub **владелец**
  (`git push` со своей машины). Локальная рабочая папка — источник правды
  (владелец собирает из неё, GitHub не обязателен для сборки).
- Нотифаер паузы (если будет игровой цикл) — **`isPaused`** (не `paused` —
  конфликт с `FlameGame.paused`).
- **Ассеты по минимуму** (владелец не рисует/озвучивает всё сам):
  графика = **эмодзи** (🍎🐶🔵⭐) + простые фигуры Flame; голос = **системный TTS**
  (`flutter_tts`, рус. подсказки/похвала — дети не читают, файлы не нужны);
  звук/джинглы = **процедурно** (Python+wave, как в танках); звуки животных — CC0
  (Freesound). Шрифт — забандленный **Unbounded** (`assets/fonts/`, кириллица;
  японский — системный фолбэк), задан как `fontFamily` темы. У игр бывают
  кастомные иконки в `assets/games/` (напр. `count_main.png` для «Счёта»).

## Зафиксированные решения (обсуждено в чате)
- **Движок:** **Flame везде** — единый Flame-канвас в каждой игре + переиспользуем
  «сок» (частицы/попапы/тряска/флэш), как в tanks/yoru.
- **Пилот:** **«Счёт»** (counting) — на нём обкатываем 4 слоя + core.
- **Механика «Счёт»:** **оба режима по наборам** — ранние наборы «считай тапом»
  (голос считает 1..N), поздние «выбери цифру».
- **Визуальных тем будет 5–10.** Палитра — `ThemeExtension` (`AppColors`) + реестр
  `AppThemes.all`; смена темы = подмена расширения. Виджеты берут цвета через
  **`context.appColors`** и **не хардкодят** значения → любая тема применяется сама.
  Финальный набор палитр — за дизайном владельца (сейчас дефолт + 2 образца).
- **Адаптивная вёрстка — без абсолютных размеров.** Размеры считаем долями от
  экрана (`LayoutBuilder`/`MediaQuery` + `clamp`), а не хардкод-пикселями — разные
  телефоны. Текст — через `textTheme` (уважает масштаб шрифта). Исключение: размер
  ассета иконки (1024², ОС сама ужимает под устройство).

## Архитектура (по образцу natsukashi_tanks)
Feature-Based. Каждая игра в `lib/features/<game>/`, **4 слоя**:
- `logic/<g>_logic.dart` — **чистая логика** (только `dart:math` + инъекция
  `Random`, 100% тестируемо), тип-исход хода для «сока». Тест: `test/<g>_logic_test.dart`.
- `game/<g>_flame_game.dart` — `FlameGame` (единый Flame-канвас): фазы, нотифаеры
  (`isPaused` и т.п.), «сок» (частицы/попапы/тряска/флэш). **Без Game Over.**
- `ui/<g>_overlays.dart` — HUD из `core/components/overlay_kit`: старт-панель,
  панель «Молодец!» со звёздами (вместо гейм-овера).
- `<g>_game_screen.dart` — экран-хост: `GameWidget` + оверлеи по фазе/паузе +
  голосовые подсказки + запись прогресса (звёзды/наборы) в `GameStorage`.
  Образцы: `features/counting/`, `features/colors_shapes/` (оба играбельны).

`core/` (портируется из tanks/yoru, перекрашивается в яркое):
- `theme` ✅ — палитра как `ThemeExtension` (`AppColors`) + реестр `AppThemes.all`
  (**51 тема**: 3 базовых + 48 портированных, `theme_palettes.dart`); `context.appColors`
  в виджетах. **Выбор темы включён** (`ThemeController` + галерея в Настройках,
  живой реколор + персист `themeId`).
- `storage/GameStorage` ✅ — открытые наборы + звёзды по играм, тема, согласие,
  настройки (звук/голос/вибро). Тесты. (Наклейки — Фаза 4.)
- `feedback/Haptics` ✅ — мягкие вибро (тап/выбор/нежный «успех»).
- `components/overlay_kit` ✅ — старт/похвала(звёзды)/пауза-панели + чипы HUD,
  адаптивные, цвета из темы. Без Game Over.
- `audio/Sfx` ✅ — фасад событий (no-op до Фазы 5).
- **`voice/Voice`** ✅ — очередь фраз; два режима: системный `flutter_tts` (`ru-RU`,
  выбор голоса в Настройках) и **встроенный пак** клипов `assets/voice/pack/`
  (audioplayers, офлайн) с фолбэком на TTS. **3 женских пака** (baya/kseniya/xenia)
  в `assets/voice/pack/<voice>/`, выбор голоса пользователем в Настройках; генерит
  `tool/gen_voice_pack.py` (реестр `Voice.packVoices`).
- **`praise/Praise`** ✅ — фразы похвалы + звёзды за набор (чистая логика + тест).
- `rating` ⏳ — оценка/почта **за родительским гейтом** (Фаза 6).
- `legal` ⏳ — документы + экран согласия (офлайн, без сбора данных → политика проста; Фаза 6).
