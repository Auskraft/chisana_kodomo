> **План, по которому идём.** Документ ведём живым: отмечаем `[x]` сделанное,
> двигаемся фазами сверху вниз. Краткий обзор проекта — в [`CLAUDE.md`](CLAUDE.md).
> Легенда статусов: ✅ готово · 🔜 следующий шаг · ⏳ позже.

## Как пользоваться
- Фазы идут **в порядке зависимостей** (нельзя строить игру без core).
- У каждой фазы: **цель**, **чек-лист** задач и **«готово, когда»** (DoD).
- Перед каждым коммитом — `flutter analyze` чисто + `flutter test` зелёные.
- Меняем план свободно: это наш рабочий инструмент, не закон.

## Сквозные принципы (всегда)
- **UX «без проигрышей»:** никаких таймеров/Game Over; ошибка → мягкое «попробуй
  ещё»; мгновенная похвала + звёзды/наклейки. Мягкий детский «сок», не аркада.
- **Адаптивная вёрстка:** размеры — доли от экрана (`LayoutBuilder`/`MediaQuery`
  + `clamp`), без абсолютных пикселей. Текст — через `textTheme`.
- **Цвета только из темы:** `context.appColors`, никаких хардкод-значений
  (готовимся к 5–10 темам, реестр `AppThemes.all`).
- **Ассеты по минимуму:** эмодзи + фигуры Flame; голос — системный TTS;
  звук — процедурно/CC0. Без рисованных ассетов до этапа бренда.
- **Движок:** Flame везде (единый канвас + переиспользуемый «сок»).
- **Пауза-нотифаер — `isPaused`** (не `paused`).
- **Git:** коммитим по ходу в `main`; пуш — в foreground (фон режет сеть).
- **Агент не собирает APK** — верификация только `analyze`/`test`; сборка у владельца.
